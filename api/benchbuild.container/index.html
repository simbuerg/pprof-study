module

benchbuild.container

</>

Container construction tool.

This tool assists in the creation of customized uchroot containers. You can define strategies and apply them on a given container base-image to have a fixed way of creating a user-space environment.

Classes
  • MockObj — Context object to be used in strategies.</>
  • ContainerStrategy — Interfaces for the different containers chosen by the experiment.</>
  • BashStrategy — The user interface for setting up a bash inside the container.</>
  • SetupPolyJITGentooStrategy — Interface of using gentoo as a container for an experiment.</>
  • Container — Manage uchroot containers.</>
  • ContainerRun — Execute commannds inside a prebuilt container.</>
  • ContainerCreate — Create a new container with a predefined strategy.</>
  • ContainerBootstrap — Check for the needed files.</>
  • ContainerList — Prints a list of the known containers.</>
Functions
  • clean_directories(builddir, in_dir, out_dir) — Remove the in and out of the container if confirmed by the user.</>
  • find_hash(container_db, key) — Find the first container in the database with the given key.</>
  • main(*args) — Main entry point for the container tool.</>
  • pack_container(in_container, out_file) — Pack a container image into a .tar.bz2 archive.</>
  • run_in_container(command, container_dir) — Run a given command inside a container.</>
  • set_input_container(_container, cfg) — Save the input for the container in the configurations.</>
  • setup_bash_in_container(builddir, _container, outfile, shell) — Setup a bash environment inside a container.</>
  • setup_container(builddir, _container) — Prepare the container and returns the path where it can be found.</>
  • setup_directories(builddir) — Create the in and out directories of the container.</>

subcommand(name, subapp=None)

</>

Registers the given sub-application as a sub-command of this one. This method can be used both as a decorator and as a normal classmethod::

@MyApp.subcommand("foo")
class FooApp(cli.Application):
    pass

Or ::

MyApp.subcommand("foo", FooApp)

.. versionadded:: 1.1

.. versionadded:: 1.3 The sub-command can also be a string, in which case it is treated as a fully-qualified class name and is imported on demand. For example,

MyApp.subcommand("foo", "fully.qualified.package.FooApp")

Create a new container with a predefined strategy.

We offer a variety of creation policies for a new container. By default a basic 'spawn a bash' policy is used. This just leaves you inside a bash that is started in the extracted container. After customization you can exit the bash and pack up the result.

classmethod

subcommand(name, subapp=None)

</>

Registers the given sub-application as a sub-command of this one. This method can be used both as a decorator and as a normal classmethod::

@MyApp.subcommand("foo")
class FooApp(cli.Application):
    pass

Or ::

MyApp.subcommand("foo", FooApp)

.. versionadded:: 1.1

.. versionadded:: 1.3 The sub-command can also be a string, in which case it is treated as a fully-qualified class name and is imported on demand. For example,

MyApp.subcommand("foo", "fully.qualified.package.FooApp")

classmethod

run(argv=None, exit=True)

</>

Runs the application, taking the arguments from sys.argv by default if nothing is passed. If exit is True (the default), the function will exit with the appropriate return code; otherwise it will return a tuple of (inst, retcode), where inst is the application instance created internally by this function and retcode is the exit code of the application.

.. note:: Setting exit to False is intendend for testing/debugging purposes only -- do not override it in other situations.

classmethod

invoke(*args, **switches)

</>

Invoke this application programmatically (as a function), in the same way run() would. There are two key differences: the return value of main() is not converted to an integer (returned as-is), and exceptions are not swallowed either.

:param args: any positional arguments for main() :param switches: command-line switches are passed as keyword arguments, e.g., foo=5 for --foo=5
